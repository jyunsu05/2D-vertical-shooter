# 2D Vertical Shooter

Unity로 제작한 2D 종스크롤 슈팅 게임 프로젝트입니다.  
플레이어 이동, 연사, 적 스폰 구역 분리, 적 타입별 이동 패턴, 피격 무적 시간, 점수 및 HP UI, 게임오버 처리까지 포함한 슈팅 게임의 핵심 루프를 직접 구현했습니다.

기존 튜토리얼 기반 구조에서 출발했지만, 현재 프로젝트에는 적 스폰 구역 분리, 적 탄환 타게팅, 플레이어 무적 시간, 적 타입별 밸런스 조정 등 커스텀 게임 로직이 추가되어 있습니다.

## 프로젝트 개요

- 장르: 2D 종스크롤 슈팅 게임
- 엔진: Unity 6
- 렌더 파이프라인: Universal Render Pipeline (URP)
- 주요 입력: 키보드 이동 + 마우스 좌클릭 발사
- 핵심 시스템:
	- 플레이어 이동 및 연속 발사
	- 적 타입별 이동 속도와 발사 패턴 분리
	- 화면 밖 적 발사 제한
	- 플레이어 피격 시 1초 무적 및 깜빡임 효과
	- 점수 / HP UI 및 게임오버 처리
	- 스폰 포인트 구역별 적 타입 제한

## 개발 환경

- Unity Editor: `6000.4.1f1`
- 주요 패키지:
	- `com.unity.render-pipelines.universal`
	- `com.unity.inputsystem`
	- `com.unity.ugui`
	- `TextMesh Pro`

## 실행 방법

1. Unity Hub에서 이 저장소 폴더를 엽니다.
2. Unity Editor 버전을 `6000.4.1f1`로 맞춥니다.
3. 메인 씬을 엽니다.
	 - 기본 게임 씬: `Assets/Scenes/SampleScene.unity`
4. Play 버튼을 눌러 실행합니다.

## 조작 방법

- 이동: `WASD` 또는 방향키 축 입력
- 공격: `마우스 좌클릭`
- 재시도: 게임오버 시 `retryButton`

## 게임 시스템 설명

### 1. 플레이어

플레이어는 화면 내부로 이동 범위가 제한되어 있으며, 마우스 좌클릭을 누르고 있는 동안 일정 간격으로 총알을 발사합니다.

플레이어 화력은 `power` 값에 따라 달라집니다.

- `power = 1`: 단일 발사
- `power = 2`: 좌우 2발 발사
- `power = 3`: 중앙 + 좌우 3발 발사

피격 시에는 즉시 HP가 감소하고, 이후 1초 동안 무적 상태가 됩니다. 이 무적 시간 동안에는 스프라이트가 깜빡여서 플레이어가 현재 무적인 상태임을 시각적으로 확인할 수 있습니다.

관련 스크립트:

- `Assets/playerBullet.cs`

### 2. 적 시스템

적은 `Enemy` 공통 스크립트를 사용하며, 프리팹 설정값에 따라 체력, 이동 속도, 총알 발사 여부, 탄환 프리팹, 발사 포인트 등을 다르게 가집니다.

현재 적 타입은 아래와 같이 구성되어 있습니다.

#### EnemyA

- 체력: 20
- 이동 속도: 3.0
- 발사 여부: 발사 안 함
- 스폰 구역: `spawn_0 ~ spawn_4`

#### EnemyB

- 체력: 30
- 이동 속도: 3.8
- 발사 여부: 발사 안 함
- 스폰 구역: `spawn_5 ~ spawn_12`

#### EnemyC

- 체력: 40
- 이동 속도: 2.4
- 발사 여부: 발사함
- 발사 간격: 2초
- 발사 포인트: 2개
- 한 번 발사 시: 2발 발사
- 스폰 구역: `spawn_0 ~ spawn_4`

EnemyC의 총알은 발사 순간 플레이어 위치를 기준으로 방향을 계산한 뒤, 그 방향으로 직선 이동합니다. 즉, 플레이어를 향해 쏘지만 발사 후에 유도되지는 않습니다.

또한 적은 화면 밖으로 나간 상태에서는 총을 발사하지 않도록 처리되어 있습니다.

관련 스크립트:

- `Assets/Enemy.cs`
- `Assets/EnemyBullet.cs`

### 3. 스폰 시스템

적 스폰은 `SpawnPoint` 스크립트에서 관리됩니다. 각 스폰 포인트는 별도 `Invoke` 호출로 재귀적으로 다시 예약되며, 일정 랜덤 딜레이 후 적을 생성합니다.

현재 스폰 규칙은 아래와 같습니다.

- 상단 구역 `spawn_0 ~ spawn_4`
	- `EnemyA`, `EnemyC`만 스폰 가능
- 측면 및 이동 구역 `spawn_5 ~ spawn_12`
	- `EnemyB`만 스폰 가능

즉, 서로 다른 스폰 구역에서는 허용되지 않은 적이 절대 생성되지 않도록 코드로 제한되어 있습니다.

또한 일부 스폰 포인트는 단순 하강이 아니라 특정 좌표를 향하는 방향 이동을 사용합니다.

관련 스크립트:

- `Assets/SpawnPoint.cs`

### 4. 점수 및 HP UI

UI 시스템은 `test.cs`에서 관리합니다.

기능:

- 현재 점수 표시
- HP 텍스트 표시
- HP 이미지 3개 표시 및 동기화
- 게임오버 패널 표시
- 재시도 버튼 처리
- 적 처치 시 점수 증가

게임오버 시에는 적 스폰을 중단하고, 현재 활성화된 적을 정리한 뒤 UI를 표시합니다.

관련 스크립트:

- `Assets/test.cs`

## 주요 구현 포인트

### 플레이어 피격 무적 처리

플레이어가 연속 피격으로 너무 빠르게 죽는 문제를 막기 위해, 피격 후 1초 무적 시간을 추가했습니다.

- 무적 중 추가 피격 무시
- 스프라이트 깜빡임 효과 적용

### 화면 밖 적 발사 제한

적이 카메라 밖으로 나간 뒤 뒤쪽에서 총을 쏘는 문제를 방지하기 위해, 현재 카메라 뷰포트 안에 있을 때만 발사 로직이 실행되도록 처리했습니다.

### 적 탄환의 비유도 타게팅

적 탄환은 단순히 아래 방향으로만 발사되지 않고, 발사 순간 플레이어 위치를 기준으로 방향을 계산합니다. 다만 유도탄처럼 플레이어를 계속 추적하지는 않습니다.

### 스폰 구역 강제 분리

스폰 포인트 인덱스를 기준으로 적 타입을 필터링해서, 원하는 적이 원하는 구역에서만 생성되도록 구현했습니다.

## 현재 확인 가능한 게임 밸런스 상태

- 플레이어는 연사 가능
- 적 A는 기본 속도로 하강하며 발사하지 않음
- 적 B는 A보다 빠르게 이동하며 발사하지 않음
- 적 C는 A보다 느리게 이동하며 2개의 발사 포인트에서 플레이어 방향으로 2발 발사
- 적 탄환은 화면 안의 적만 발사 가능
- 플레이어는 피격 후 1초 무적
